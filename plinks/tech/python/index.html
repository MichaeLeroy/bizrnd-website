Clean Code Reload

Code imported into a Python IDE runs as if unmodified in that session when the source code has been changed. This behavior results because the interpreter uses the byte compiled code without revisiting the source. I find this behavior to be an impediment to Python's generally rapid development cycle and was glad to see a good discussion of this issue and its solutions in "Learning Python" by Mark Lutz.

The conventional solution to this difficulty is to follow import statements with reloads to ensure that the environment picks up source changes subsequent to the initial import.
